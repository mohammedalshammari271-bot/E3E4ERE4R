# 🌐 إعدادات GitHub Pages لتفعيل النشر التلقائي

لتفعيل نشر موقعك بنجاح من خلال GitHub Actions، يرجى اتباع الخطوات البسيطة التالية في مستودع الكود الخاص بك على GitHub:

1. افتح مستودع الكود الخاص بك على **GitHub**.
2. اذهب إلى التبويب **Settings** (الإعدادات) في القائمة العلوية.
3. من القائمة الجانبية اليسرى، اختر **Pages**.
4. تحت قسم **Build and deployment** (البناء والنشر)، ابحث عن خيار **Source** (المصدر).
5. قم بتغيير القيمة المحددة من `Deploy from a branch` إلى **GitHub Actions**.

---

### 💡 لماذا هذه الخطوة مهمة؟
الافتراضي في بعض المستودعات هو النشر من فرع (`Deploy from a branch`)، ولكن نظرًا لأننا قمنا بإنشاء بيئة بناء مؤتمتة ذكية ومتكاملة باستخدام سير عمل GitHub Actions المخصص، يحتاج GitHub لتفويض Actions للقيام بعملية النشر وإدارة الـ Artifacts تلقائيًا.

بمجرد تغيير هذا الخيار، سيعمل سير العمل المرفق في المستودع (`.github/workflows/deploy.yml`) تلقائيًا مع كل عملية دفع (`Push`) جديدة لفرع `main` لينشر التطبيق بنجاح!
